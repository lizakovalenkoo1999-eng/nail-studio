# PRD (Product Requirements Document)
# Сайт маникюрного салона "Nail Studio"

---

## 1. Введение

### 1.1 Назначение документа
Данный документ описывает требования к веб-сайту маникюрного салона "Nail Studio". PRD определяет функциональные и нефункциональные требования, пользовательские сценарии, а также критерии успешного завершения проекта.

### 1.2 Общее видение продукта
**Nail Studio** — современный, элегантный сайт маникюрного салона, который:
- Привлекает новых клиентов через профессиональный визуальный стиль
- Обеспечивает удобную онлайн-запись на услуги
- Демонстрирует портфолио работ мастеров
- Формирует доверие через отзывы и информацию о салоне

### 1.3 Целевая аудитория
| Сегмент | Описание | Возраст |
|---------|----------|---------|
| Основной | Женщины, интересующиеся уходом за ногтями | 20-45 лет |
| Вторичный | Мужчины (маникюр для бизнеса) | 25-55 лет |
| Дополнительный | Подростки (эксперименты с дизайном) | 15-19 лет |

### 1.4 Бизнес-цели
- **Конверсия:** Увеличение количества онлайн-записей на 40% в первый год
- **Привлечение:** Рост органического трафика на 60% через SEO
- **Удержание:** Повторные визиты через программу лояльности
- **Репутация:** Формирование имиджа премиум-салона

## 3. Функциональные требования

### 3.1 Главная страница (Homepage)

**FR-1.1** Сайт должен отображать hero-секцию с:
- Заголовком (слоган салона)
- Подзаголовком с кратким описанием
- Кнопкой CTA "Записаться онлайн"
- Анимированным фоном (видео/параллакс)

**FR-1.2** Сайт должен отображать секцию "Преимущества" с:
- 4-6 карточками с иконками
- Текстовым описанием каждого преимущества
- Анимацией появления при скролле

**FR-1.3** Сайт должен содержать быстрые ссылки на:
- Услуги
- Галерею работ
- Онлайн-запись
- Контакты

### 3.2 Раздел "Услуги" (Services)

**FR-2.1** Сайт должен отображать каталог услуг с:
- Табами категорий (Маникюр, Педикюр, Наращивание, Дизайн)
- Карточками услуг с названием, описанием, ценой, длительностью
- Кнопкой записи на конкретную услугу

**FR-2.2** Система табов должна:
- Позволять переключение без перезагрузки страницы
- Запоминать выбранную категорию
- Иметь анимацию переключения

**FR-2.3** Карточка услуги должна содержать:
- Название услуги
- Краткое описание (1-2 предложения)
- Цену (руб.)
- Длительность (мин.)
- Кнопку "Записаться"

### 3.3 Галерея работ (Portfolio)

**FR-3.1** Галерея должна поддерживать:
- Отображение работ в сетке (masonry или grid)
- Фильтрацию по категориям (дизайн, маникюр, педикюр)
- Пагинацию или бесконечный скролл

**FR-3.2** При клике на работу должен открываться lightbox с:
- Увеличенным изображением
- Навигацией (следующее/предыдущее)
- Описанием работы (опционально)
- Кнопкой закрытия

**FR-3.3** Галерея должна иметь:
- Hover-эффекты на превью
- Lazy loading изображений
- Анимацию загрузки

### 3.4 Раздел "О салоне" (About)

**FR-4.1** Страница "О нас" должна содержать:
- Историю салона
- Философию/миссию
- Фото интерьера
- Информацию о материалах и оборудовании

**FR-4.2** Сайт должен отображать статистику салона:
- Количество довольных клиентов
- Лет опыта
- Количество мастеров
- Выполненных работ
- С анимацией счётчика при скролле

### 3.5 Раздел "Мастера" (Team)

**FR-5.1** Карточка мастера должна содержать:
- Фото
- Имя и фамилию
- Специализацию
- Опыт работы (лет)
- Кнопку записи к этому мастеру
- Ссылки на соцсети

**FR-5.2** Список мастеров должен:
- Отображаться в виде сетки карточек
- Иметь фильтрацию по специализации

### 3.6 Раздел "Отзывы" (Reviews)

**FR-6.1** Слайдер отзывов должен:
- Автоматически прокручиваться
- Иметь ручное управление (стрелки, точки)
- Отображать: фото клиента, имя, текст отзыва, дату, рейтинг

**FR-6.2** Сайт должен поддерживать:
- Отображение рейтинга в звёздах
- Возможность оставить отзыв (форма)
- Интеграцию с внешними платформами (Google, Яндекс)

### 3.7 Онлайн-запись (Booking)

**FR-7.1** Форма записи должна содержать поля:
- Имя клиента (обязательно)
- Телефон (обязательно)
- Email (опционально)
- Выбор услуги (выпадающий список)
- Выбор мастера (опционально)
- Желаемая дата
- Желаемое время
- Комментарий

**FR-7.2** Система записи должна:
- Валидировать данные в реальном времени
- Показывать доступное время
- Отправлять подтверждение на email
- Показывать успешное сообщение после отправки

**FR-7.3** UX требования:
- Floating label для полей ввода
- Анимация фокуса полей
- Состояния загрузки и успеха
- Адаптивная версия для мобильных

### 3.8 Контакты (Contact)

**FR-8.1** Секция контактов должна содержать:
- Адрес с картой (Google/Yandex Maps)
- Телефон с возможностью клика (tel:)
- Email с возможностью клика (mailto:)
- Время работы
- Соцсети (Instagram, Telegram, WhatsApp)

**FR-8.2** Карта должна:
- Показывать местоположение салона
- Быть интерактивной
- Содержать маркер с названием

### 3.9 Админ-панель

**FR-9.1** CMS должна поддерживать:
- Управление услугами (CRUD)
- Управление мастерами (CRUD)
- Управление галереей (CRUD)
- Управление отзывами
- Управление записями
- Редактирование контента страниц

**FR-9.2** Требования к CMS:
- Интуитивный интерфейс
- Загрузка изображений с сжатием
- Предпросмотр изменений

## 4. Нефункциональные требования

### 4.1 Производительность

| Метрика | Целевое значение |
|---------|------------------|
| First Contentful Paint (FCP) | < 1.5 сек |
| Largest Contentful Paint (LCP) | < 2.5 сек |
| Time to Interactive (TTI) | < 3.5 сек |
| Cumulative Layout Shift (CLS) | < 0.1 |
| Mobile Speed Score | > 90 |

### 4.2 Адаптивность (Responsive)

**FR-10.1** Сайт должен корректно отображаться на:
- Desktop (1920px+)
- Laptop (1280px - 1919px)
- Tablet (768px - 1279px)
- Mobile (320px - 767px)

**FR-10.2** Мобильные требования:
- Touch-friendly элементы (min 44x44px)
- Hamburger-меню
- Адаптивные изображения
- Отсутствие горизонтального скролла

### 4.3 Доступность (Accessibility)

| Стандарт | Уровень | Требования |
|----------|---------|------------|
| WCAG 2.1 | AA | Соответствие |
| ADA | Level II | Соответствие |

**Требования:**
- Alt-тексты для всех изображений
- ARIA-атрибуты для интерактивных элементов
- Keyboard navigation
- Focus states для всех элементов
- Color contrast ratio >= 4.5:1

### 4.4 SEO

**FR-11.1** Техническое SEO:
- Semantic HTML5 теги
- Meta title и description для каждой страницы
- Schema.org разметка (LocalBusiness, Product, Review)
- XML sitemap
- robots.txt
- Open Graph теги
- Twitter Card теги
- Канонические URL

**FR-11.2** On-page SEO:
- Один H1 на странице
- Структурированные заголовки (H2-H6)
- Оптимизированные изображения (alt, name)
- Внутренние ссылки

### 4.5 Безопасность

- HTTPS (SSL сертификат)
- Защита от XSS
- Защита от CSRF
- Валидация всех данных на сервере
- Безопасная обработка форм
- Rate limiting для API

### 4.6 Браузерная поддержка

| Браузер | Минимальная версия |
|---------|-------------------|
| Chrome | 90+ |
| Firefox | 88+ |
| Safari | 14+ |
| Edge | 90+ |
